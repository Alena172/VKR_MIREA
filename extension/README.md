# Browser Extension

## Обзор

Браузерное расширение - это легкий клиент для захвата слов в ContextVocab. Его задача - сократить путь от "я увидел незнакомое слово на странице" до "это слово уже находится в моей учебной системе".

Основные возможности:

- читать выделенный текст с активной вкладки
- запрашивать перевод у backend
- сохранять захваченное слово или фразу в словарь пользователя

Расширение намеренно остается небольшим и сфокусированным. Основной учебный опыт по-прежнему находится во frontend.

## Технологии

- Chrome / Edge Manifest V3 extension
- работа через popup-интерфейс
- интеграция с backend API по HTTP

## Что умеет сейчас

Сейчас расширение поддерживает два основных сценария:

1. Перевести выделенный текст через backend translation API.
2. Сохранить результат в словарь через capture endpoint.

То есть это вспомогательный инструмент для захвата новой лексики, а не полноценный интерфейс обучения.

## Установка

1. Открой `chrome://extensions` или `edge://extensions`.
2. Включи Developer Mode.
3. Нажми `Load unpacked`.
4. Выбери папку [extension](d:\VKR\VKR_V3_Curs\extension).

## Требования к запуску

Расширение ожидает локально запущенный backend, сейчас ориентированный на:

- `http://localhost:8000`

Также требуется аутентифицированный пользователь, потому что сохранение слов жестко привязано к пользовательскому контексту.

## Модель аутентификации

Popup получает JWT по email пользователя и затем использует этот токен для авторизованных запросов.

Текущее поведение:

- получение токена происходит прямо из popup-flow
- токен хранится в browser extension storage
- расширение использует `/auth/me`, чтобы при необходимости восстановить текущий user context

Это практичная dev-схема, и к ней стоит относиться именно так.

## Используемые backend-endpoints

Расширение зависит от:

- `POST /api/v1/translate/me`
- `POST /api/v1/vocabulary/me/from-capture`
- `GET /api/v1/auth/me`

Если адрес backend изменится, расширение тоже нужно будет обновить. Пока этот адрес остается ориентированным на dev-среду и недостаточно хорошо параметризован для аккуратного production-развертывания.

## Роль в продукте

Расширение стоит воспринимать как быстрый инструмент захвата:

- обнаружить слово в живом контексте
- перевести его с учетом контекста
- добавить его в учебную систему

Тренировка, повторение, история и большая часть рекомендательных сценариев остаются задачами web-приложения.
