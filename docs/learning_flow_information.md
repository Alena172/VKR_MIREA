# IDEF0-модель: персонализированное изучение английской лексики

## Назначение документа

Документ описывает систему в нотации `IDEF0` на уровне:

- контекстной диаграммы `A-0`;
- декомпозиции верхнего уровня `A0`.

На текущем этапе сознательно проектируются только **функции и информационные потоки**. Поля **управления** и **механизмов** не детализируются.

---

## 1. Границы системы

Система должна обеспечивать единый цикл персонализированного изучения английской лексики:

1. принять языковой материал, с которым столкнулся пользователь;
2. включить его в персональную модель изучаемой лексики;
3. сформировать учебную активность;
4. оценить результат и обновить состояние освоения.

С точки зрения предметной области это соответствует реальному сценарию системы:

- пользователь приносит слово и контекст;
- система формирует персональную словарную и семантическую запись;
- система предлагает, что изучать и что повторять;
- система выдаёт задание;
- пользователь отвечает;
- система возвращает обратную связь и обновлённый прогресс.

---

## 2. Контекстная диаграмма A-0

### A-0. Обеспечить персонализированное изучение английской лексики

### Входы A-0

- `Новое слово и пример его использования`  
  Слово или выражение, выбранное пользователем, вместе с фразой или предложением, где оно встретилось.
- `Ответы пользователя на задания`  
  Ответы при изучении и повторении слов, включая правильные и ошибочные попытки.

### Выходы A-0

- `Персональные материалы для изучения`  
  Подобранные слова, задания на изучение и повторение, а также слова для следующего шага обучения.
- `Обратная связь по выполнению`  
  Результаты оценки, пояснения ошибок, рекомендации по улучшению ответа.
- `Обновлённый прогресс в изучении слов`  
  Текущий результат обучения, проблемные слова и приоритеты для повторения.

### Текстовое представление A-0

```text
                 ┌───────────────────────────────────────────────┐
Новое слово      │                                               │   Персональные
и пример его ───►│ A-0 Обеспечить персонализированное изучение   ├──► материалы
использования    │ английской лексики                            │   для изучения
                 │                                               ├──► Обратная связь
Ответы          ─►│                                               │   по выполнению
пользователя     │                                               ├──► Обновлённый прогресс
                 └───────────────────────────────────────────────┘   в изучении слов
```

### Пояснение по корректности IDEF0

Контекстная диаграмма содержит:

- **одну функцию верхнего уровня**;
- **входы слева**;
- **выходы справа**;
- единый результат деятельности системы без смешения с внутренними подсистемами.

Это соответствует правилам IDEF0 для уровня `A-0`.

---

## 3. Декомпозиция A0

Для декомпозиции верхнего уровня система разделена на 4 связанные функции:

- `A1` — зарегистрировать учебный языковой материал;
- `A2` — сформировать персональную модель изучаемой лексики;
- `A3` — сформировать персонализированную учебную активность;
- `A4` — оценить результат обучения и актуализировать состояние освоения.

Такое разбиение логично и соответствует реальному жизненному циклу данных в системе:  
**новое слово → список слов пользователя для изучения → задание → ответ → обновлённый прогресс**.

## 5. Подробное описание блоков декомпозиции

## A1. Зарегистрировать новое слово для изучения

### Назначение

Преобразовать новое слово, которое пользователь встретил, в запись, пригодную для дальнейшего изучения.

### Входы A1

- `Выбранное слово или выражение`
- `Фраза или предложение с этим словом`
- `Место, где пользователь встретил это слово`

### Выходы A1

- `Подготовленное слово для изучения`
- `Сохранённый пример использования`
- `Запись о новом слове`

### Смысл потоков

- `Подготовленное слово для изучения` передаётся в `A2` как основа для дальнейшей работы системы.
- `Сохранённый пример использования` передаётся в `A2`, чтобы система учитывала смысл слова в конкретной фразе.
- `Запись о новом слове` передаётся в `A2` как начало личной истории изучения этого слова.

---

## A2. Сформировать список слов пользователя для изучения и повторения

### Назначение

Добавить новое слово в список слов, которые пользователь изучает, понять, как оно связано с уже знакомыми словами, и определить, нужно ли учить его сейчас или повторять позже.

### Входы A2

- `Подготовленное слово для изучения` из `A1`
- `Сохранённый пример использования` из `A1`
- `Запись о новом слове` из `A1`
- `Сведения о словах, которые даются легко или трудно` из `A4`

### Выходы A2

- `Список слов пользователя для изучения`
- `Понятный смысл слова в данном контексте`
- `Слова, которые сейчас важнее изучить`
- `Слова, которые уже пора повторить`

### Смысл потоков

- `Список слов пользователя для изучения` передаётся в `A3` как перечень слов, которые уже добавлены пользователем в обучение.
- `Понятный смысл слова в данном контексте` передаётся в `A3`, чтобы задание проверяло именно то значение, с которым столкнулся пользователь.
- `Слова, которые сейчас важнее изучить` передаются в `A3` для выбора нового материала.
- `Слова, которые уже пора повторить` передаются в `A3` для формирования повторения.
- `Сведения о словах, которые даются легко или трудно`, возвращающиеся из `A4`, помогают заново расставить приоритеты.

---

## A3. Сформировать персональное учебное задание

### Назначение

Преобразовать состояние личного словаря пользователя в конкретные задания для изучения и повторения.

### Входы A3

- `Список слов пользователя для изучения` из `A2`
- `Понятный смысл слова в данном контексте` из `A2`
- `Слова, которые сейчас важнее изучить` из `A2`
- `Слова, которые уже пора повторить` из `A2`

### Выходы A3

- `Персональные материалы для изучения`
- `Задания для пользователя`
- `Правильные ответы`
- `Слова, вошедшие в задание`

### Смысл потоков

- `Персональные материалы для изучения` являются внешним выходом системы на уровне `A0`.
- `Задания для пользователя` передаются пользователю для выполнения.
- `Правильные ответы` передаются в `A4`, потому что без них нельзя оценить результат.
- `Слова, вошедшие в задание` передаются в `A4`, чтобы обновление прогресса относилось к конкретным словам.

---

## A4. Оценить результат обучения и обновить прогресс изучения

### Назначение

Сопоставить ответы пользователя с заданиями, определить результат обучения и обновить сведения о том, какие слова усвоены хорошо, а какие вызывают трудности.

### Входы A4

- `Ответы пользователя на задания`
- `Задания для пользователя` из `A3`
- `Правильные ответы` из `A3`
- `Слова, вошедшие в задание` из `A3`

### Выходы A4

- `Обратная связь по выполнению`
- `Обновлённый прогресс в изучении слов`
- `Сведения о словах, которые даются легко или трудно`

### Смысл потоков

- `Обратная связь по выполнению` возвращается пользователю как внешний выход системы.
- `Обновлённый прогресс в изучении слов` является внешним выходом системы и показывает текущее состояние обучения.
- `Сведения о словах, которые даются легко или трудно` возвращаются в `A2` и используются для следующего цикла персонализации.

## 6. Связность декомпозиции как единой системы

Чтобы декомпозиция соответствовала правилам IDEF0, блоки должны образовывать непрерывную цепочку преобразования входов в выходы. В данной модели это обеспечивается следующим образом:

- `A1` не выдаёт конечный результат, а только подготавливает учебно значимые данные;
- `A2` не создаёт задания напрямую, а формирует персональную модель, из которой они должны выводиться;
- `A3` не оценивает ответы, а только формирует предъявляемый учебный материал и эталоны;
- `A4` завершает цикл, выдаёт внешние результаты и возвращает внутренний адаптационный сигнал в `A2`.

Таким образом, система работает как единый замкнутый контур:

```text
Новое слово
   → A1 Регистрация
   → A2 Список слов пользователя
   → A3 Учебное задание
   → A4 Оценка и обновление прогресса
   → Обновлённый прогресс
   → A2 Новая персонализация следующего цикла
```

## 7. Проверка соответствия реальности системы

Предложенная IDEF0-модель совпадает с фактической логикой проектируемой системы:

- система действительно начинается с появления нового слова или выражения;
- далее это слово включается в личный словарь пользователя;
- на основе персонального состояния формируются задания и рекомендации;
- по результатам ответов пересчитывается прогресс и меняются дальнейшие приоритеты изучения.

При этом модель остаётся **правильной с точки зрения IDEF0**, потому что:

- функции описаны как преобразования, а не как инструменты;
- потоки названы как информационные сущности;
- декомпозиция идёт от общего назначения к связанным подфункциям;
- на диаграмме нет привязки к конкретным технологиям, API, БД или программным модулям.
